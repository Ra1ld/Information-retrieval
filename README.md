# Information Retrieval System

This repository contains a comprehensive Information Retrieval (IR) system built in Python. The system includes a web crawler for fetching data from Wikipedia, text preprocessing modules, the creation of an inverted index, and the implementation of several classic and modern retrieval models for searching the collected documents.

## Features

*   **Web Crawler**: A configurable web crawler to fetch and parse articles from Wikipedia, starting from a given seed URL.
*   **Text Preprocessing**: A complete text processing pipeline including tokenization, stop-word removal, case conversion, and lemmatization using NLTK.
*   **Inverted Index**: Efficient creation and storage of an inverted index mapping terms to the documents in which they appear.
*   **Multiple Retrieval Models**: Implementation and comparison of four different retrieval models:
    *   Boolean Retrieval
    *   TF-IDF (Term Frequency-Inverse Document Frequency)
    *   Vector Space Model (using Cosine Similarity)
    *   Okapi BM25
*   **Model Evaluation**: The retrieval models are evaluated using the standard `20 Newsgroups` dataset to measure performance metrics like Precision, Recall, F1-score, and MAP (Mean Average Precision).

## How It Works

The project follows a sequential pipeline from data collection to search and evaluation.

1.  **Crawling**: The script begins by crawling Wikipedia, starting from the [Web scraping](https://en.wikipedia.org/wiki/Web_scraping) page. It collects text content from up to 20 pages within a maximum depth of 2. The collected URLs and their corresponding text are saved into `wikipedia_data.csv`.

2.  **Preprocessing**: The raw text from `wikipedia_data.csv` is loaded. Each document's text undergoes a cleaning process where special characters are removed, the text is tokenized into words, converted to lowercase, stopwords are filtered out, and words are reduced to their base form (lemmatized).

3.  **Indexing**: An inverted index is constructed from the preprocessed tokens. This index is a dictionary where each key is a unique term (word) and its value is a list of document IDs (indices) where the term appears. The final index is saved as `inverted_index.json`.

4.  **Searching & Ranking**: A predefined query, `"crawler AND bot"`, is used to search the indexed documents. The query is preprocessed, and each of the four retrieval models is used to fetch and rank the relevant documents from the collection.
    *   **Boolean**: Retrieves documents matching the strict boolean logic.
    *   **TF-IDF, VSR, BM25**: Retrieve and rank documents based on their calculated relevance scores for the given query.

5.  **Evaluation**: The performance of the retrieval models is assessed using the `20 Newsgroups` dataset. A sample query `"space shuttle launch"` is used to retrieve relevant documents, and standard IR metrics are calculated and printed.

## Files in This Repository

*   `Source code.py`: The main Python script that executes the entire pipeline, from crawling to searching and evaluation.
*   `Commented Code.ipynb`: A Jupyter Notebook version of the script with extensive, step-by-step explanations of the code and concepts (written in Greek).
*   `wikipedia_data.csv`: The dataset generated by the web crawler, containing the URLs and text from scraped Wikipedia articles.
*   `inverted_index.json`: The inverted index created from `wikipedia_data.csv`, stored in JSON format.

## How to Run

### Prerequisites

You need Python 3 and the following libraries:
*   `requests`
*   `beautifulsoup4`
*   `pandas`
*   `nltk`
*   `scikit-learn`
*   `numpy`

### Installation

1.  Install the required packages using pip:
    ```bash
    pip install requests beautifulsoup4 pandas nltk scikit-learn numpy
    ```

2.  Run the Python interpreter and download the necessary NLTK data:
    ```python
    import nltk
    nltk.download('punkt')
    nltk.download('stopwords')
    nltk.download('wordnet')
    nltk.download('omw-1.4')
    ```

### Execution

To run the full process, simply execute the `Source code.py` script:
```bash
python "Source code.py"
```
The script will first perform the crawling and indexing, saving the `wikipedia_data.csv` and `inverted_index.json` files. It will then proceed to run the search queries using all implemented models and print the ranked results, followed by the evaluation on the 20 Newsgroups dataset.
